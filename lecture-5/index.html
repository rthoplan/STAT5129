INNER JOIN

The INNER JOIN returns only the rows that have matching values in both tables. It's the most common type of join and is perfect for finding related data.

SQL Query

SELECT
    c.Name,
    o.OrderID,
    o.OrderDate
FROM
    Customers c
INNER JOIN
    Orders o ON c.CustomerID = o.CustomerID;

LEFT JOIN

The LEFT JOIN returns all rows from the left table, and the matching rows from the right table. If there is no match, the columns from the right table will have NULL values.

RIGHT JOIN

The RIGHT JOIN returns all rows from the right table, and the matching rows from the left table. If there is no match, the columns from the left table will have NULL values. This can be used to find "orphaned" records in the right table that have no corresponding entry in the left one.

FULL OUTER JOIN

A FULL OUTER JOIN returns all rows from both tables, with NULL values where a match does not exist. It's a combination of a LEFT and a RIGHT JOIN.

SQL Query (MySQL Emulation)

(SELECT c.Name, o.OrderID, o.OrderDate
FROM Customers c LEFT JOIN Orders o
ON c.CustomerID = o.CustomerID)
UNION
(SELECT c.Name, o.OrderID, o.OrderDate
FROM Customers c RIGHT JOIN Orders o
ON c.CustomerID = o.CustomerID);

CROSS JOIN

A CROSS JOIN produces a Cartesian product of the two tables. This means every row from the first table is combined with every row from the second table. This is very useful for scenarios like creating a master list of all possible combinations (e.g., matching every product with every store), but it can be dangerous with large tables.
