The Problem: Reading Row-by-Row

JSON and CSV are row-based. To find all values in one column (e.g., `eventType`), the computer must read through every single row from top to bottom. This is slow for big data.

Imagine we want to find how many "add_to_cart" events happened. Watch how much data we have to scan.

Step 2: Parquet (The High-Performance Warehouse üè¢)

Parquet is columnar. It stores all values for a single column together. It's not human-readable but is extremely fast for analytics.
